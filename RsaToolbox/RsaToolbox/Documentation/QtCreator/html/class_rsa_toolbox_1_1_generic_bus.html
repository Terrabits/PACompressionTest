Detailed Description

GenericBus is a pure virtual base class for instrument connection methods.

This GenericBus class is pure virtual and must be implemented in a subclass. See VisaBus and TcpBus for examples.

The GenericBus class should be used to add new connection methods to RsaToolbox. When doing so, all pure virtual methods need to be implemented, the appropriate GenericBus constructors should be called, and, when reimplementing non-virtual functions, the equivalent base class implementation should called.

The pure virtual methods that require reimplementation are

public:

  • virtual bool isOpen() const
  • virtual bool read(char *buffer, uint bufferSize_B)
  • virtual bool write(QString scpi)
  • virtual bool binaryRead(char *buffer, int bufferSize_B, uint &bytesRead)
  • virtual bool binaryWrite(QByteArray scpi)
  • virtual QString status() const

    public slots:
  • virtual bool lock()
  • virtual bool unlock()
  • virtual bool local()
  • virtual bool remote()

See the documentation for each method for specific details and hints.

See also
VisaBus, TcpBus

Constructor & Destructor Documentation

GenericBus ( QObject *  parent = 0)
explicit

The default constructor.

The default constructor initializes a GenericBus object that is not connected to anything.

That is: connectionType() returns NO_CONNECTION address() returns "" and isOpen is false.

Default constructors in base classes should call this first.

Calling read and write functions on a disconnected bus will cause timeouts and other errors to occur. A pointer to a NoBus instance can be safely used as a placeholder in the event your code may try to use it anyway.

Note
As a general rule, the RsaToolbox::NO_CONNECTION value should be used when a connection is not or cannot be established, although this is not enforced by GenericBus. Always check isOpen and isClosed for a definitive answer.

QByteArray binaryRead ( )

Reads binary response from the instrument.

The data is read and returned as-is; no assumptions of content or string terminations are made. An example use-case would be in transferring the raw, arbitrary contents of a file.

Note
Partial reads are allowed, and occur when bufferSize_B is less than the total read size. In this case, binaryRead should be called until all data is read. It is the responsibility of the callee to reassemble the data correctly.

bool read ( char *  buffer,
uint  bufferSize_B 
)
pure virtual

Reads from instrument using c-style character buffer.

This method is pure virtual and must be overloaded in base class implementations. This function should call printRead internally.

This function assumes that the read value is a c-style string that is null terminated. As such, it is assumed that the data to be read does not contain invalid or reserved values (such as the null character). binaryRead does not make this assumption.

A failed read should return false and return an empty c-string:

buffer[0] = '\0';
Note
GenericBus uses this function, along with an internal buffer, to implement other higher-level read functions.
Parameters
bufferC-style character buffer
bufferSize_BSize of buffer, in bytes
Returns
true if read is successful

Implemented in VisaBus, TcpBus, and NoBus.

QString read ( )

Reads C-string compatible data from the instrument.

Also note that bufferSize_B() should be greater than or equal to the expected read size (including null termination). If you want to perform a read without these limitations, (for example, when reading raw binary data in multiple transactions) use binaryRead instead.

bool write ( QString  scpi)
pure virtual

Writes scpi to the connected instrument.

This method is pure virtual and must be overloaded in base class implementations.

If in implementing this function a C-style const char* representation of scpi is necessary (in a US-ASCII/Latin-1 character set) it can be retrieved by calling:

scpi.toLatin1().constData()

It is important to recognize that the above pointer only exists ephemerally, during the execution of this line of code. This is because const char* is pointing to an internal buffer in QByteArray, which disappears after this line is executed.

If you need a persistent pointer, use the verbose snippet below:

QByteArray byteArray = scpi.toLatin1();
const char* cstring = byteArray.constData();
Note
bufferSize_B does not limit write comamnds, as the internal buffer is not used.

true if successul

Implemented in VisaBus, TcpBus, and NoBus.
