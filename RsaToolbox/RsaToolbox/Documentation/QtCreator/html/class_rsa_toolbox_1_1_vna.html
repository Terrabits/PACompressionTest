Detailed Description

The Vna class provides an interface for controlling a Rohde & Schwarz Vector Network Analyzer.

Vna is derived from the GenericInstrument class, and as such has the ability to connect to a Vna using a GenericBus subclass such as VisaBus. It also can connect to a Log for SCPI logging.

In addition to the required basic commands implemented in GenericInstrument, Vna implements most of the functionality specific to Rohde & Schwarz VNAs. It does so in a style and manner that is consistent with Qt's style guidelines and native data types.

Vna is organized into parts that represent each of the logical elements of R&S VNAs, including:

Vna::fileSystem() Vna::channel() Vna::trace() Vna::diagram()

As an example, the channels can be queried by calling Vna::channels(). Each channel can then be manipulated through the Vna::channel() interface.

Convenience functions are also provided that allow you to determine the properties of a specific Vna. These can be found in: Vna::properties()

Some examples include: VnaProperties::isZnbFamily() determines whether or not you are currently controlling a ZNB.

VnaProperties::minimumFrequency_Hz() determines the minimum frequency of the instrument.

VnaProperties::hasSourceAttenuators() determines if the source attenuator option is installed on the instrument.

Global VNA settings can be found through the Vna::settings() interface. These settings include things like user-preset, emulation modes and port power limits. These settings effect all channels and traces.

See also
GenericInstrument, GenericBus, Log
