Detailed Description

CalStandard represents an RF calibration standard and its associated properties

A CalStandard can be one or two port, of different connector types and genders, it can be VNA port-specific, it can be defined by a touchstone file or by a lumped-element model, and is also limited to a specific frequency range.

A CalStandard can be given a label (optional) to uniquely identify it.

Note
In order to prevent ambiguity, Rohde & Schwarz VNAs require standards with two ports to order connector genders and vna ports a specific way. For example, a mixed gender thru standard must be defined such that:
calStandard.gender1() == MALE_GENDER;
calStandard.gender2() == FEMALE_GENDER;
Similarly, port specific standards must list the lower-numbered port as port1 and the higher-numbered port as port2. While CalStandard does not enforce this, failure to abide by these guidelines will cause instrument errors when using the RsaToolbox::Vna CalKit interface.

For a list of supported standard types, see RsaToolbox::CalStandardType.

See also
RsaToolbox::CalStandardType, Connector, Vna::CalKit(), Vna::Calibration

bool isSameModel ( const VnaCalStandard &  other) const

Compares the lumped element model of two standards.

If the standards being compared are of a different type, or either standard is not defined by a lumped element model, this method returns false.

If the standards are of the same type and are both defined by model, the model parameters are compared for equality and the result is returned.
