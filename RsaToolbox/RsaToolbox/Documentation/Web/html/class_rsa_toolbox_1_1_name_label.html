Detailed Description

The NameLabel class handles identifiers that include both a name and a label.

NameLabel is primarily a data type for resources that have both a name and a label as identifiers. For example, a Vector Network Analyzer may have two calibration kits registered that have the same name, but have different serial numbers that can be applied as labels. A NameLabel is then used to handle the identifier without having to perform the extra business logic to handle two identifier strings.

The example below iterates through each calibration kit registered with the Vna object (vna) looking for cal kits that contain a Male-to-Male thru standard:

...
QVector<NameLabel> calKits = vna.GetCalibrationKits();
foreach(NameLabel kit, calKits) {
// Do something
}

Note: NameLabels are case-sensitive. Although the name and the label may not independently be unique, the combination of the two must be unique to a particular resource.

Parses NameLabels out of list while ignoring instances of ignore.

list must follow a particular format. For example, for separator = ",": list = (name1, label1, name2, label2, ...)

A typical Rohde & Schwarz VNA response, for example:
result = "name1,'label1',name2,'label2',..."
can be parsed with the following snippet:

NameLabel::parse(result, ",", "\'");
